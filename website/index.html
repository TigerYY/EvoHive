TEAM AGENT WORKSPACE

让团队进化为智能体

EvoHive 是一个面向小团队的团队智能体工作台，让能力共享、知识沉淀与持续协作在同一套系统中完成。

POSITIONING

不是另一个聊天机器人，而是团队的智能体工作台

在清晰边界下装配 Agent、复用 Skill、沉淀知识，并让团队能力持续进化。

不是

  • 单一聊天助手
  • 全公司统一大脑
  • 无限自治的黑盒系统

是

  • 团队级智能体运行环境
  • 团队能力共享与装配平台
  • 受监督、可进化、可回滚的协作系统

EvoHive 面向 3–20 人扁平小团队：运行时按团队隔离，能力按团队共享与跨团队发布，不共享原始会话与原始记忆。

CORE MODEL

双层模型

Team Workspace 与 Capability Exchange 构成产品内核。

Team Workspace

团队工作台

以团队为边界的运行时与资源容器：Agents、会话、工作区存储、共享技能库、知识视图、权限与审计。

Capability Exchange

能力交换层

团队间能力发布、发现、安装与治理：Skill / Workflow / Policy Pack / Agent Blueprint，版本与兼容性、Owner 与风险级别。

SIX-TUPLE

六元组能力映射

完整保留六层：出入口、路由、频道、技能、记忆、沙箱，在小团队场景下收敛而非删层。

Entry Points

统一协议转换，1–2 主入口（Web + Chat），入站事件带 teamId / channel

Routing

来源 / 角色 / 任务 / 默认四类规则，显式兜底与可观测日志

Channels

Team Workspace + 稳定 Session Key，会话与记忆按 Key 隔离

Skills

声明式能力单元，团队内共享 + 能力交换层安装，effectivePolicy 不扩权

Memory

Compaction + 关键事实，团队内长期记忆；跨团队仅 Knowledge View

Sandbox

分级隔离，最小权限、审计日志、Policy Pack，安装技能不继承全部权限

DEPLOYMENT

部署模式

支持私有环境、客户自租云主机、平台多租户 SaaS、混合部署。

私有环境 / 独立设备

客户侧控制面与数据面，内网或外网 LLM 自配；平台提供安装包与文档。

客户自租云主机

单租户，镜像或 Helm 部署；可选平台升级与监控。

平台多租户 SaaS

控制面与运行面平台托管，按租户逻辑隔离，快速开通。

混合部署

敏感数据在客户侧，控制面与能力目录可上云，统一治理。

INTERACTION

交互方式

Web Team Console、Chat Entry、API/Webhook、审批中心、Run Log / Ops Dashboard。

  • 团队成员：会话与任务、成果查看、转人工、知识检索
  • 团队管理员：Agent 装配、权限、知识视图、审批、审计
  • 能力发布者：发布 Skill / Workflow / Blueprint / Policy
  • 运营 / Owner：使用分析、成本归因、版本治理、回滚

OUTPUTS

成果与输出

即时结果、资产化成果、持续动作、运营与治理产出，均有沉淀与归属规则。

即时结果

回复、结构化结果、表单、审批单 → 会话与审计

资产化成果

代码库、文档库、知识包、Workflow、Blueprint → 工作区与 Capability Exchange

持续动作

定时巡检、推广执行、跟进任务 → 可审计、可关闭

EVOLUTION

受监督的自进化

观测 → 反馈 → 诊断 → 提案 → 评测 → 审批 → 灰度 → 晋级或回滚；三层门禁（发布前、安装时、运行时）。

可演化对象：Skill、Workflow、Blueprint、Policy Pack、Knowledge View、Routing Pack、Memory Pack。禁止自动扩权、自动跨团队共享 raw memory、自动改生产代码直接上线。

ROADMAP

阶段路线

MVP

私有部署优先；Web Console + Chat Entry；Skill 级发布与安装；文档/代码/任务输出。

V1

客户云主机部署；审批中心、能力目录、Run Log；Workflow、Blueprint、Knowledge View；成本归因。

V2

多租户 SaaS + 混合部署；跨团队能力网络；Routing Pack、Memory Pack；签名发布、风险扫描。
